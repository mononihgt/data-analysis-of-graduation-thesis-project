# Documentation Guidelines

## Scope
This directory holds durable methodology notes, architecture notes, development workflow, and analysis rationale.

## Conventions
- Use lowercase kebab-case filenames, for example `pdtask-main-effect.md`.
- Prefer concise docs that point to scripts, outputs, or checks instead of duplicating implementation details.
- Keep statistical design constraints in docs and reference them from scripts or reports when they affect interpretation.
- Update `README.md` and root `AGENTS.md` when adding a new canonical workflow document.
